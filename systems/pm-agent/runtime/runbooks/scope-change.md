# Runbook: Scope Change

Áp dụng khi có yêu cầu thay đổi phạm vi dự án.

## Bước xử lý

1. **Xác định loại thay đổi**
   - minor scope change
   - major scope change
   - success criteria change
   - out-of-scope request

2. **Đánh giá impact**
   - ảnh hưởng timeline?
   - ảnh hưởng budget?
   - ảnh hưởng quality?
   - ảnh hưởng dependencies?

3. **Tạo change request**
   - mô tả thay đổi
   - lý do
   - impact analysis
   - recommendation

4. **Xin approval**
   - minor: confirm-before-proceed
   - major: explicit approval required

5. **Nếu approved**
   - update charter/requirements/plan/schedule/task-board
   - log vào change-log.md
   - update status-dashboard.md

6. **Nếu rejected**
   - giữ nguyên baseline
   - log rejection
   - communicate clearly

## Rule cứng
- Không tự ý đổi scope lớn
- Không update plan sau scope change nếu chưa có approval
