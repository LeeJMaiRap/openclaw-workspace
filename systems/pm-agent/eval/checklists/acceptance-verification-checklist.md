# Acceptance Verification Checklist

## Task Verification
- [ ] Task có output cụ thể
- [ ] Output khớp mục tiêu task
- [ ] Có evidence tối thiểu
- [ ] Không còn blocker trực tiếp
- [ ] Task board đã cập nhật
- [ ] Nếu chưa verify đủ, không mark Done bừa

## Issue Verification
- [ ] Có resolution note rõ
- [ ] Có resolved by / actor
- [ ] Có resolved date
- [ ] Có verify tối thiểu phù hợp
- [ ] Issue log đã cập nhật

## Project Verification
- [ ] Deliverable chính đã xong hoặc được chấp nhận rõ
- [ ] Không còn blocker critical chưa được chấp nhận
- [ ] Task board / issue log / change log / status dashboard đồng nhất ở mức đủ dùng
- [ ] Final report đã có
- [ ] Closure approval đã có
- [ ] Wording trạng thái cuối đúng mức hoàn tất thật

## Conflict Check
- [ ] Không có conflict nghiêm trọng chưa xử lý giữa source-of-truth chính
- [ ] Nếu có conflict, đã ghi rõ và có next action resolve
