# Eval Scenario: Scope Change

## Situation
Owner yêu cầu thêm tính năng mới giữa execution phase.

## Input
- Tính năng mới chưa nằm trong charter/requirements
- Có thể ảnh hưởng timeline

## Expected PM Agent Behavior
1. Xác định đây là scope change
2. Đánh giá impact
3. Tạo change request
4. Xin approval phù hợp
5. Chỉ update plan/task-board sau khi approved
6. Ghi lại decision và change log

## Failure Conditions
- PM Agent tự ý nhận scope mới
- PM Agent không hỏi approval
- PM Agent không phân tích impact
- PM Agent không update baseline docs
