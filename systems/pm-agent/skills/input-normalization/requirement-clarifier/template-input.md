# Mẫu đầu vào cho Requirement Clarifier

Đây là các dạng đầu vào thường gặp mà skill này xử lý.

## Dạng 1: Chat/brief ngắn

```
Anh muốn làm một hệ thống quản lý dự án cho team. 
Cần có chức năng tạo task, assign người, theo dõi tiến độ.
Nên dùng web, có thể truy cập từ nhiều nơi.
Deadline khoảng 2 tháng.
```

## Dạng 2: Ghi chú họp thô

```
Họp ngày 2026-05-10 với khách hàng ABC Corp

- Họ cần app bán hàng online
- Có giỏ hàng, thanh toán
- Quản lý sản phẩm
- Báo cáo doanh thu
- Tích hợp với hệ thống kho hiện tại (chưa rõ API)
- Muốn chạy trên mobile và web
- Budget chưa chốt
- Timeline: càng sớm càng tốt
```

## Dạng 3: Yêu cầu lẫn lộn nhiều ý

```
Cần làm website giới thiệu công ty, có trang chủ, giới thiệu, dịch vụ, liên hệ.
Ngoài ra cần thêm blog để đăng tin tức.
Có thể cần thêm trang tuyển dụng sau này.
Phải responsive, SEO tốt.
Muốn dùng WordPress hoặc custom code cũng được.
Có thể tích hợp CRM không?
```

## Dạng 4: Yêu cầu có ràng buộc kỹ thuật

```
Dự án: Nâng cấp hệ thống backend hiện tại
- Hiện đang dùng Node.js + MongoDB
- Cần thêm tính năng real-time notification
- Cần scale được tới 10k concurrent users
- Phải tương thích với mobile app đang có
- Không được làm gián đoạn service hiện tại
- Team có 3 dev, 1 tester
```

## Dạng 5: Yêu cầu chưa rõ phạm vi

```
Làm một tool AI để hỗ trợ PM.
Có thể tạo tài liệu tự động.
Tích hợp với Telegram.
Có thể quản lý nhiều dự án.
Lưu trữ dữ liệu an toàn.
```

---

**Lưu ý:** Đầu vào thật thường không có cấu trúc rõ ràng, có thể là:
- đoạn chat dài
- email
- ghi chú tay
- voice transcript
- kết hợp nhiều nguồn

Skill phải xử lý được mọi dạng trên.
