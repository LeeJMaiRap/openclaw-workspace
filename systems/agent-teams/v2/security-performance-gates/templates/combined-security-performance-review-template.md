# Phase 71 — Combined Security / Performance Review Template v2

## Use When

Use when one phase requests both security and performance claims.

## Required Split

```text
Security evidence must use security matrix rows.
Performance evidence must use benchmark matrix rows.
Shared risks must use risk severity rubric.
Accepted residual risks must use risk acceptance form.
```

## Combined Review Record

```text
Project / phase:
Project type pack:
Requested security claim:
Requested performance claim:
Security reviewer:
Performance reviewer:
PM owner:
Verification Level:
```


## Performance Result

```text
Matrix rows completed:
Commands executed:
Benchmark results:
Highest severity:
Claim allowed:
Claim forbidden:
Risk acceptance links:
```

## PM Recommendation

```text
Accept / Reject / Needs Review / Blocked
Allowed user-facing claims:
Forbidden user-facing claims:
Follow-up owners:
```

## Stop Conditions

```text
Critical risk in either track
High risk without required sign-off
security claim lacks security matrix evidence
performance claim lacks benchmark output
production/deploy/browser/dependency approval needed but missing
```
