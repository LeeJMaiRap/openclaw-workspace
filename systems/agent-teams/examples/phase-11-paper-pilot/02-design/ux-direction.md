# UX Direction — Issue Tracker MVP

## Screens

```text
- Landing
- Issue List
- Issue Detail
- Create Issue
- Owner/Admin status controls
```

## States Required

```text
Landing: default
Issue List: loading, empty, error, populated, filtered
Issue Detail: loading, not found, error, populated
Create Issue: default, validation error, submit success, submit error
Status Update: idle, saving, success, failure
```

## Layout Direction

```text
- mobile-first
- simple navigation: Home / Issues / Create
- issue status badges
- priority badges
- clear primary action
```

## Accessibility Notes

```text
- labels for all form fields
- visible focus states
- semantic buttons/links
- status changes announced in page text
```
