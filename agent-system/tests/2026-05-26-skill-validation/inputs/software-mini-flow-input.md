# Software Mini Flow Input

## Scenario

Build a tiny internal task board for a 1-person team.

## Requirements

- User can create task with title, description, status, priority.
- User can list tasks.
- User can update status: todo, doing, done.
- User can filter by status.
- No login required for MVP.
- Local-only demo acceptable.

## Constraints

- Prefer Next.js + TypeScript style.
- Keep MVP small.
- Do not install packages.
- Create planning/design artifacts only, not production codebase.

## Acceptance

- PRD clear enough for PM/Architect handoff.
- Plan has tasks, dependencies, done criteria.
- Frontend spec names components/states.
- Backend spec names endpoints/contracts.
- QA report can say PASS/FAIL/BLOCKED for artifact readiness.
