# Architecture Skill Test Input

## Scenario

Design architecture for tiny internal task board MVP.

## Product requirements

- create task with title, description, status, priority
- list tasks
- update status: todo, doing, done
- filter by status
- no login for MVP
- local demo acceptable

## Constraints

- Prefer Next.js + TypeScript style.
- No dependency install.
- No production deploy.
- Keep MVP small and reversible.
- Need handoff notes for Frontend, Backend, QA.

## Expected output

- Architecture brief.
- ADRs.
- module boundaries.
- API/data notes.
- risks.
- specialist handoff.
- verification level and missing evidence.
