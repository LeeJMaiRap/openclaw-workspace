# Quy tắc lưu trữ cho PM Agent

## Mục tiêu
Quy định rõ thông tin nào không cần lưu, thông tin nào lưu vào memory, thông tin nào lưu vào Obsidian, và thông tin nào cần lưu cả hai.

## Nguyên tắc tổng quát
- Memory dùng cho ngữ cảnh vận hành, preference, quy tắc cộng tác, lesson learned và facts ngắn cần truy hồi nhanh.
- Obsidian dùng cho tài liệu dự án, note có cấu trúc, decision log, project knowledge và artifacts PM lâu dài.
- Chat Telegram chỉ là nguồn vào, không phải source of truth.
- Không lưu mọi thứ; chỉ lưu khi có giá trị dùng lại hoặc ảnh hưởng tới bước tiếp theo.

## Không cần lưu
Không lưu các nội dung:
- chào hỏi, xã giao
- xác nhận ngắn như “ok”, “ừ”, “tiếp đi”
- câu hỏi nhất thời không có giá trị dài hạn
- output/suy luận trung gian
- lỗi nhỏ không tạo lesson learned

## Lưu vào memory
Lưu vào memory khi thông tin là:
- preference hoặc quy tắc làm việc của anh Doanh
- ngữ cảnh vận hành quan trọng
- trạng thái roadmap/task đang dở
- lesson learned của agent
- fact ngắn cần truy hồi nhanh

Ví dụ:
- 1-2 ý thì trả lời trực tiếp, từ 3 ý trở lên thì liệt kê trước
- cần kiểm tra file/lệnh trước khi kết luận
- Telegram là kênh giao tiếp chính
- roadmap đang ở Bước 1.1 / 1.2

## Lưu vào Obsidian
Lưu vào Obsidian khi thông tin là:
- Project Charter, PRD, BRD
- Meeting Minutes
- Action Items
- Decision Log
- Risk Register
- Status Report
- project summary
- stakeholder notes
- assumptions / constraints
- user stories / acceptance criteria
- tri thức dự án có cấu trúc

## Lưu cả memory và Obsidian
Lưu cả hai khi thông tin vừa quan trọng cho vận hành hiện tại, vừa là tri thức lâu dài.

Quy tắc:
- Obsidian giữ bản đầy đủ/source of truth.
- Memory chỉ giữ tóm tắt truy hồi nhanh.

Ví dụ:
- quyết định lớn về workflow PM Agent
- meeting quan trọng có nhiều action items
- roadmap chính thức của project

## Khi nào phải hỏi anh trước
Phải hỏi trước khi:
- thông tin nhạy cảm hoặc riêng tư
- chưa chắc nên lưu vào memory hay Obsidian
- ý tưởng chưa được chốt
- tạo note/tài liệu lớn
- thay đổi cấu trúc kho tri thức

Không cần hỏi trước khi:
- anh nói rõ “nhớ điều này” hoặc “lưu lại”
- đó là quy tắc làm việc rõ ràng
- đó là decision đã chốt
- đó là lesson learned rõ ràng

## Rule set nhanh
- Quy tắc cộng tác / preference / lesson learned / ngữ cảnh vận hành → Memory
- Tài liệu PM / note dự án / decision log / project knowledge → Obsidian
- Quyết định lớn ảnh hưởng cả vận hành và dự án → Obsidian + memory tóm tắt
- Chat ngắn / xác nhận / nội dung tạm → Không lưu
- Chưa chắc → Hỏi anh trước
