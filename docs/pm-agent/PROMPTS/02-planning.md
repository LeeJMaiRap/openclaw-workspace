# Prompt 2: Lập kế hoạch dự án

## Input
- Charter đã được xác nhận
- Requirements đã rõ ràng
- Scope đã chốt

## Nhiệm vụ

### Bước 1: Tạo Spec
Tạo file `projects/[project-name]/02-planning/spec.md` trả lời:
- Phải xây cái gì?
- Chức năng nào?
- Hành vi mong muốn?
- User flow?
- Acceptance criteria?

### Bước 2: Tạo Plan
Tạo file `projects/[project-name]/02-planning/plan.md` trả lời:
- Sẽ triển khai như thế nào?
- Chia thành bao nhiêu phase?
- Mốc nào?
- Thứ tự ưu tiên?

### Bước 3: Tạo Rules
Tạo file `projects/[project-name]/02-planning/rules.md` định nghĩa:
- Tiêu chuẩn code
- Tiêu chuẩn tài liệu
- Quy trình git
- Quy trình testing
- Quality gates
- Change management

### Bước 4: Tạo WBS
Tạo file `projects/[project-name]/02-planning/wbs.md` phân rã:
- Level 1: Dự án
- Level 2: Module/Phase
- Level 3: Feature
- Level 4: Task

Mỗi task cần:
- ID
- Tên
- Mô tả
- Estimate
- Dependencies
- Assignee (PM Agent hoặc Worker)

### Bước 5: Tạo Schedule
Tạo file `projects/[project-name]/02-planning/schedule.md` với:
- Milestones
- Timeline
- Critical path
- Gantt chart (markdown table)

### Bước 6: Tạo Risk Register
Tạo file `projects/[project-name]/02-planning/risk-register.md` với:
- Danh sách rủi ro
- Xác suất & mức độ ảnh hưởng
- Mitigation plan
- Risk matrix

### Bước 7: Báo cáo lại
Trình bày:
- Bộ tài liệu planning đã tạo
- Tổng số task
- Timeline dự kiến
- Rủi ro chính
- Kế hoạch thực thi tiếp theo

## Output
- spec.md
- plan.md
- rules.md
- wbs.md
- schedule.md
- risk-register.md
- Sẵn sàng chuyển sang execution
