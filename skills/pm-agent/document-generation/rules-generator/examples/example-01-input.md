# Ví dụ 1: Input cho rules hệ thống quản lý task

## Nguồn: Project Spec + Project Plan

### Stack kỹ thuật
- Frontend: React.js
- Backend: Node.js + Express
- Database: MongoDB
- Authentication: JWT
- Styling: Tailwind CSS

### Bối cảnh thực thi
- Team nội bộ nhỏ
- Mục tiêu là MVP trong khoảng 2 tháng
- Cần dùng nội bộ trước, chưa phải production quy mô lớn
- Cần tránh lỗi do code không nhất quán và phân quyền sai

### Yêu cầu nổi bật
- Cần coding standards rõ ràng
- Cần git workflow đơn giản nhưng đủ chặt
- Cần manual testing cho MVP
- Cần quality gates trước khi merge
- Cần change management để tránh scope creep
