## Các lỗi thực tế đã gặp và bài học ghi nhớ

### Lỗi 1 - nhầm file entry frontend
- Đã từng tạo `frontend/index.html` là trang demo tĩnh, không phải entry chuẩn của Vite
- Hậu quả: user luôn thấy giao diện demo tĩnh thay vì app React thật
- Cách sửa: trả `index.html` về dạng chuẩn có `<div id="root"></div>` và script `src/main.jsx`

### Lỗi 2 - icon import sai từ `lucide-react`
- `Footer.jsx` import `Facebook`, `Twitter`, `Instagram` nhưng package hiện tại không export các icon đó
- Hậu quả: frontend trắng trang do runtime error
- Cách sửa: thay bằng icon có thật như `Globe`, `Send`, `Camera`

### Lỗi 3 - mismatch category giữa frontend và backend
- Backend seed/categories dùng: `electronics`, `fashion`, `food`, `books`, `home`, `sports`, `other`
- Frontend dùng: `electronics`, `fashion`, `books`, `home`, `beauty`, `sports`
- Hậu quả: `CATEGORY_META[product.category]` bị `undefined`, crash tại `Products.jsx`
- Cách sửa: đồng bộ category, thêm `food`, `other`, bỏ `beauty`, và thêm fallback an toàn

### Lỗi 4 - hướng dẫn sai do suy đoán quá sớm
- Đã có lúc trả lời dựa trên giả định thay vì rà source/log/browser console thật
- User phản hồi bực vì token/time bị mất vô ích
- Bài học: phải quay lại kiểm tra tận gốc trước khi hướng dẫn tiếp
